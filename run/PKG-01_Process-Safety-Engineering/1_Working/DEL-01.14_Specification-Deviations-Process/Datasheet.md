## 4. Construction

### 4.1 Register Structure (Typical)

| Column | Description |
|--------|-------------|
| Deviation ID | Unique identifier (format: SD-PR-###) |
| Date Raised | Date deviation identified |
| Description | Clear description of deviation |
| TM Spec Reference | Standard/specification being deviated from |
| Requirement | Original requirement text |
| Proposed Alternative | Proposed deviation approach |
| Justification | Technical/commercial justification |
| Risk Assessment | Impact assessment (safety, reliability, cost, schedule) using Low/Medium/High scale |
| Status | Submitted / Under Review / Approved / Rejected |
| TM Approval Ref | TM approval document reference |
| Approval Date | Date of TM approval |
| Conditions | Any conditions attached to approval |
| Related Documents | P&IDs, calculations, etc. affected |
| Responsible Engineer | Engineering Contractor contact |
| Remarks | Additional notes |

### 4.2 Content Requirements

The Deviation Register shall include:
- All process-related deviations from TM standards identified during design
- Complete justification for each deviation
- Risk assessment for each deviation
- TM approval status and reference
- Traceability to affected design documents
