# Procedure: DEL-01.01 Process Design Basis / DBM (Design Basis & Design Criteria)

**Document ID:** DEL-01.01-PR
**Revision:** Phase 2.2 (INITIALIZED)
**Date:** 2026-02-01
**Status:** INITIALIZED

---

## Purpose

### Procedure Objective

This procedure defines the step-by-step process for developing, reviewing, and issuing the Process Design Basis / Design Basis Memorandum (DBM) for the Puget Sound Optimization Project. It ensures consistent development aligned with SOW requirements and TM expectations.

### Scope

This procedure covers:
- DBM development from initiation through IFC issue
- Internal review and approval processes
- TM (client) review coordination
- Revision and update management

## Steps

### Phase 1: Preparation and Planning

| Step | Action | Responsible | Output |
|------|--------|-------------|--------|
| 1.1 | Review SOW requirements for DBM (§3.1.3.1, §4.2.2.1-§4.2.2.3) | Process Lead | Requirements checklist |
| 1.2 | Review preliminary DBM and appendices (B, C, A-1, A-2) | Process Lead | Annotated review notes |
| 1.3 | Identify gaps and open questions requiring TM clarification (e.g., blending range discrepancy: SOW 20%-80% vs DBM 10%-100% NEAT) | Process Lead | Open items log |
| 1.4 | Submit clarification requests to TM via agreed interface | Project Manager | RFI/Query log entries |
| 1.5 | Request TMP Standards (Attachment A08) from TM | Project Engineer | Standards package |
| 1.6 | Obtain document number from TM Document Control | Document Controller | Assigned document number |
| 1.7 | Establish DBM development schedule aligned with FEED milestones | Process Lead | DBM schedule |

### Phase 2: Data Gathering and Validation

| Step | Action | Responsible | Output |
|------|--------|-------------|--------|
| 2.1 | Compile design basis parameters from preliminary DBM | Process Engineer | Parameter compilation |
| 2.2 | Validate hydraulic data against Appendix C | Process Engineer | Validated hydraulic parameters |
| 2.3 | Confirm site conditions from layout appendices (A-1, A-2) | Process Engineer | Site conditions summary |
| 2.4 | Obtain fluid property data (crude specifications) | Process Engineer | Fluid property tables |
| 2.5 | Compile applicable codes and standards list (CER OPR, CSA Z662, PHMSA 195, TMP) | Process Engineer | Standards register |
| 2.6 | Verify applicability of provincial (BC) and state (Washington) regulations | Process Engineer | Regulatory applicability confirmation |
| 2.7 | Obtain TMP material standards and design criteria (Attachment A08) | Process Engineer | TMP standards package |
| 2.8 | Receive TM responses to clarification requests | Process Lead | Confirmed parameters |
| 2.9 | Document confirmed blending range (SOW vs DBM resolution; clarify NEAT terminology) | Process Lead | Blending basis documented |
| 2.10 | Establish design life (typically 25-30 years) and corrosion allowances | Process Engineer | Design life and corrosion basis |
| 2.11 | Determine minimum turndown flow rate requirements | Process Engineer | Turndown requirements |

### Phase 3: Document Development (IFR)

| Step | Action | Responsible | Output |
|------|--------|-------------|--------|
| 3.1 | Prepare DBM outline per guidance document structure | Process Lead | DBM outline |
| 3.2 | Draft Section 1: Introduction and Project Overview | Process Engineer | Draft Section 1 |
| 3.3 | Draft Section 2: Scope and Boundaries (include battery limits definition and Laurel exclusion) | Process Engineer | Draft Section 2 |
| 3.4 | Draft Section 3: Codes and Standards | Process Engineer | Draft Section 3 |
| 3.5 | Draft Section 4: Process Design Criteria (flows, fluids, pressures) | Process Engineer | Draft Section 4 |
| 3.6 | Draft Section 5: Equipment Design Basis (5 VS6 pumps, 5x6 manifold [5 pumps × 6 discharge connections], VFDs, flow meters, turndown requirements) | Process Engineer | Draft Section 5 |
| 3.7 | Draft Section 6: Environmental Conditions (ambient, seismic, site-specific) | Process Engineer | Draft Section 6 |
| 3.8 | Draft Section 7: Material Selection Criteria (preliminary, based on TMP standards and fluid compatibility) | Process Engineer | Draft Section 7 |
| 3.9 | Draft Section 8: Interface Conditions with existing Sumas Terminal | Process Engineer | Draft Section 8 |
| 3.10 | Draft Section 9: Safety and Overpressure Protection Philosophy | Safety Engineer | Draft Section 9 |
| 3.11 | Draft Section 10: Deviations from TMP Standards (if any) with approval status | Process Engineer | Draft Section 10 |
| 3.12 | Document all assumptions with basis and required confirmations | Process Engineer | Assumptions register |
| 3.13 | Verify all TMP standard deviations are identified and documented | Process Lead | Deviation log |
| 3.14 | Perform internal technical review | Process Lead | Review comments |
| 3.15 | Incorporate internal review comments | Process Engineer | Updated draft |
| 3.16 | Perform interdisciplinary review (Mechanical, Piping, I&C, Electrical) | IDC Coordinator | IDC comments |
| 3.17 | Incorporate IDC comments | Process Engineer | IFR draft |
| 3.18 | Issue DBM for TM review (IFR) | Document Controller | IFR submission |

### Phase 4: Review Cycle and IFD Issuance

| Step | Action | Responsible | Output |
|------|--------|-------------|--------|
| 4.1 | Receive TM review comments | Project Manager | Comment register |
| 4.2 | Prepare comment responses | Process Lead | Response register |
| 4.3 | Incorporate agreed changes into DBM | Process Engineer | Updated DBM |
| 4.4 | Resolve any outstanding clarifications with TM | Process Lead | Closed items |
| 4.5 | Perform final internal review | Process Lead | Approval |
| 4.6 | Issue DBM as IFD (end of FEED) | Document Controller | IFD submission |

### Phase 5: Detailed Design Updates and IFC Issuance

| Step | Action | Responsible | Output |
|------|--------|-------------|--------|
| 5.1 | Incorporate HAZOP findings affecting design basis (per DEL-01.15) | Process Lead | Updated parameters |
| 5.2 | Incorporate vendor data affecting design basis | Process Engineer | Updated equipment basis |
| 5.3 | Update DBM for model review and change notice impacts | Process Engineer | Updated DBM |
| 5.4 | Perform interdisciplinary verification | IDC Coordinator | Verification record |
| 5.5 | Issue DBM as IFC | Document Controller | IFC submission |

## Verification

### Quality Checks

| Check | Criteria | Method | Timing |
|-------|----------|--------|--------|
| Completeness | All required sections populated (including battery limits, material selection, deviations) | Checklist review | Before IFR |
| Consistency | Parameters align across sections and with PFDs/P&IDs/sizing calcs | Cross-reference check | Before IFR |
| Blending Range Clarity | Blending range discrepancy (SOW vs DBM) resolved and terminology clarified (NEAT defined) | Review and TM confirmation | Before IFD |
| TBD Resolution | All TBDs addressed or approved | TBD log review | Before IFD |
| Standard Compliance | Meets TMP standards; deviations identified and approved | TM review | IFR/IFD |
| Assumption Tracking | All critical assumptions confirmed or flagged | Assumptions register review | Before IFD |
| Regulatory Alignment | Provincial and state regulatory applicability confirmed; no non-compliances without approved deviation | Standards compliance check | Before IFD |
| Preliminary DBM Verification | All preliminary DBM parameters verified per SOW §4.2.2.2 requirement | Parameter comparison | Before IFR |
