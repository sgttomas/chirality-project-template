# Procedure: Piping Data Books

## 1. Purpose

### 1.1 Objective
This procedure defines the steps for compiling, reviewing, and issuing Piping Data Books for the Puget Sound Optimization Project.

### 1.2 Scope
Applicable to all piping within the PSO project scope at Sumas Tank Farm.

## 2. Prerequisites

### 2.1 Input Documents
| Document | Source | Status Required |
|----------|--------|-----------------|
| Piping Isometrics (As-Built) | DEL-02.13 | As-Built |
| Hydrotest Packages | DEL-02.21 | Complete with certificates |
| Material Certifications | Fabrication Contractor | Complete |
| Welding Records | Fabrication/Installation Contractor | Complete |

## 3. Steps

### 3.1 Preparation

| Step | Action | Responsible |
|------|--------|-------------|
| 3.1.1 | Confirm data book organization method with TM | Piping Lead |
| 3.1.2 | Establish data book boundaries (system/area) | Piping Engineer |
| 3.1.3 | Prepare data book template with section structure | Document Controller |
| 3.1.4 | Set up document tracking register | Document Controller |

### 3.2 Progressive Collection (During Construction)

| Step | Action | Responsible |
|------|--------|-------------|
| 3.2.1 | Collect material certifications at material receipt | QC Representative |
| 3.2.2 | Collect welding records progressively during fabrication | QC Representative |
| 3.2.3 | Collect NDE records upon completion of each report | QC Representative |
| 3.2.4 | Collect pressure test certificates after each test | QC Representative |
| 3.2.5 | Update document tracking register | Document Controller |
| 3.2.6 | Identify and resolve missing documentation | Document Controller |

### 3.3 As-Built Documentation

| Step | Action | Responsible |
|------|--------|-------------|
| 3.3.1 | Obtain as-built red-line markups from construction | Piping Engineer |
| 3.3.2 | Incorporate markups into final drawings | Piping Designer |
| 3.3.3 | Issue as-built drawings for data book inclusion | Document Controller |
| 3.3.4 | Verify as-built dimensions for critical items | Piping Engineer |

### 3.4 Compilation and Review

| Step | Action | Responsible |
|------|--------|-------------|
| 3.4.1 | Organize all documents per data book structure | Document Controller |
| 3.4.2 | Create table of contents with page references | Document Controller |
| 3.4.3 | Verify material traceability chains | QC Representative |
| 3.4.4 | Verify weld-to-NDE cross-references | QC Representative |
| 3.4.5 | Review for completeness against checklist | Piping Engineer |
| 3.4.6 | Compile PDF with bookmarks | Document Controller |

### 3.5 Approval and Issue

| Step | Action | Responsible |
|------|--------|-------------|
| 3.5.1 | Submit draft data book for internal review | Document Controller |
| 3.5.2 | Incorporate review comments | Document Controller |
| 3.5.3 | Submit to TM for review | Document Controller |
| 3.5.4 | Address TM comments | Piping Engineer |
| 3.5.5 | Obtain final approval signatures | Piping Lead |
| 3.5.6 | Issue final data book | Document Controller |
| 3.5.7 | Archive electronic and hard copy versions | Document Controller |

## 4. Verification

### 4.1 Quality Checks

| Check | Criteria | Method |
|-------|----------|--------|
| Completeness | All sections per template present | Checklist review |
| Traceability | Material certs trace to installation | Sample audit (10%) |
| Accuracy | As-built drawings match field | Spot verification |
| Legibility | All documents readable | Visual review |
| Signatures | All QC records properly signed | 100% review |
