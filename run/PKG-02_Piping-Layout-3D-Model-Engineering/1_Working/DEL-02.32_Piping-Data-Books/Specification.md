# Specification: Piping Data Books

## 1. Scope

### 1.1 Purpose
This specification defines the requirements for compiling and issuing Piping Data Books for the Puget Sound Optimization Project, capturing all design, fabrication, installation, and testing documentation for piping systems.

### 1.2 Applicability
- All new piping associated with booster pump installation
- Modified existing piping systems
- Piping for 5x6 manifold system
- Tie-in connections to existing systems

### 1.3 Exclusions
- Instrument tubing (covered under Instrumentation package)
- Vendor package internal piping documentation (included in vendor data books)
- Electrical conduit

## 2. Requirements

### 2.1 Content Requirements
The Piping Data Books shall include:

| Requirement | Description |
|-------------|-------------|
| REQ-01 | Cover sheet with project identification and approvals |
| REQ-02 | Complete table of contents with page references |
| REQ-03 | Design documentation (criteria, calculations, specifications) |
| REQ-04 | As-built piping drawings (isometrics, GAs, key plans) |
| REQ-05 | Material certifications with full traceability |
| REQ-06 | Complete welding documentation (WPS, PQR, welder quals, weld logs) |
| REQ-07 | NDE inspection records and reports |
| REQ-08 | Pressure test certificates and records |
| REQ-09 | Coating and insulation inspection records |
| REQ-10 | Approved specification deviations |
| REQ-11 | Resolved punch list documentation |

### 2.2 Format Requirements
- Compiled as indexed PDF with bookmarks
- Hard copy binders with tabbed sections (**ASSUMPTION**)
- Consistent document numbering throughout
- All documents legible at 100% scale
- Native files provided separately where required by SOW

### 2.3 Quality Requirements
- All documents shall be final approved versions
- All QC records shall be signed and dated
- Material certifications shall be traceable to installed material
- Test records shall reference specific test packages

### 2.4 Traceability Requirements
- Heat numbers traceable from certification to installation location
- Weld numbers traceable to welder and WPS
- NDE reports traceable to specific welds
- Test records traceable to specific systems/lines

## 4. Verification

### 4.1 Verification Methods
| Method | Description |
|--------|-------------|
| Completeness Check | Verify all required sections present |
| Cross-Reference | Verify material certs match installation records |
| Audit | Sample audit of traceability chains |
| Review | Technical review prior to turnover |

### 4.2 Acceptance Criteria
- 100% of required sections complete
- All QC records properly executed
- Material traceability verified for critical items
- No outstanding punch items
- TM review and approval

## 5. Documentation

### 5.1 Deliverable Format
