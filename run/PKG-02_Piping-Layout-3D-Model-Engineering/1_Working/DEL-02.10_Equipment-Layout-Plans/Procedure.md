# Procedure: DEL-02.10 Equipment Layout Plans

## 1. Purpose
This procedure defines the step-by-step process for developing the Equipment Layout Plan drawing(s) for the Puget Sound Optimization Project (PSO) at Sumas Tank Farm.

## 2. Scope
This procedure applies to the development, review, and issuance of equipment layout plan drawings from FEED through Detailed Design phases, including updates for vendor data incorporation.

## 3. Responsibilities

| Role | Responsibilities |
|------|-----------------|
| Piping Lead | Overall deliverable ownership, technical review, approval |
| Piping Designer/Engineer | Drawing development, coordination, revisions |
| CAD Technician | Drawing production per standards |
| Mechanical Lead | Equipment data coordination, vendor data management |
| Civil/Structural Lead | Foundation interface coordination |
| Electrical Lead | Motor and VFD coordination |
| Procurement | Vendor data transmittal |
| Document Controller | Document numbering, revision control, issuance |

## 4. Prerequisites

Before commencing drawing development:

## 5. Procedure

### Step 1: Gather Input Data
**Responsible:** Piping Designer/Engineer

| Action | Details |
|--------|---------|
| 1.1 | Obtain equipment list (DEL-03.01) for equipment identification |
| 1.2 | Obtain preliminary equipment datasheets (DEL-03.02) |
| 1.3 | Obtain preliminary/budgetary equipment dimensional data |
| 1.4 | Obtain area plans (DEL-02.09) for layout context |
| 1.5 | Obtain 3D model extract for current arrangement |
| 1.6 | Obtain TM CAD standards and drawing templates |
| 1.7 | Establish vendor data tracking log |
| 1.8 | Document any gaps in input data |

**Output:** Collected input data package, vendor data tracking log

---

### Step 2: Develop Equipment List for Layouts
**Responsible:** Piping Designer/Engineer, Mechanical Lead

| Action | Details |
|--------|---------|
| 2.1 | Identify equipment requiring layout drawings |
| 2.2 | Group equipment logically (pumps, manifold, sump system) |
| 2.3 | Determine number of layout sheets required |
| 2.4 | Establish equipment layout sheet assignments |
| 2.5 | Document vendor data requirements for each equipment |

**Output:** Equipment layout sheet plan

---

### Step 3: Set Up Drawing Files
**Responsible:** CAD Technician, Piping Designer

| Action | Details |
|--------|---------|
| 3.1 | Set up drawing files per TM CAD standards |
| 3.2 | Import site coordinate grid |
| 3.3 | Set up appropriate scale (typically 1:50) |
| 3.4 | Set up layer structure for equipment layouts |
| 3.5 | Prepare title block with preliminary document numbers |
| 3.6 | Establish nozzle schedule table format |

**Output:** Drawing templates ready for content

---

### Step 4: Develop Preliminary Equipment Layouts
**Responsible:** CAD Technician, Piping Designer

| Action | Details |
|--------|---------|
| 4.1 | Import/draft equipment outlines from preliminary data |
| 4.2 | Position equipment per 3D model arrangement |
| 4.3 | Add equipment tags and descriptions |
| 4.4 | Show preliminary nozzle locations |
| 4.5 | Add preliminary anchor bolt patterns (if available) |
| 4.6 | Add reference dimensions to site grid |
| 4.7 | Mark drawings as "PRELIMINARY - SUBJECT TO VENDOR DATA" |

**Output:** Preliminary equipment layout drawings

---

### Step 5: Add Support Elements
**Responsible:** CAD Technician, Piping Designer

| Action | Details |
|--------|---------|
| 5.1 | Add driver (motor) locations and orientations |
| 5.2 | Add auxiliary equipment locations (lube systems, etc.) |
| 5.3 | Add baseplate outlines |
| 5.4 | Reference foundation drawings |
| 5.5 | Add maintenance clearance zones |
| 5.6 | Add sections/elevations as required for clarity |

**Output:** Drawings with support elements

---

### Step 6: Develop Nozzle Schedules
**Responsible:** Piping Designer

| Action | Details |
|--------|---------|
| 6.1 | List all equipment nozzles from P&IDs |
| 6.2 | Add nozzle identification per vendor data |
| 6.3 | Add nozzle size, rating, facing |
| 6.4 | Add nozzle orientation (degrees or direction) |
| 6.5 | Add nozzle elevation references |
| 6.6 | Include nozzle schedule table on drawing |

**Output:** Nozzle schedules complete

---

### Step 7: Complete Drawing Annotations
**Responsible:** CAD Technician, Piping Designer

| Action | Details |
|--------|---------|
| 7.1 | Add all required dimensions |
| 7.2 | Add drawing references and cross-references |
| 7.3 | Complete legend with all symbols |
| 7.4 | Add general notes |
| 7.5 | Add north arrow and scale notation |
| 7.6 | Complete title block information |

**Output:** Annotated draft drawings

---

### Step 8: Discipline Coordination
**Responsible:** Piping Designer, Design Coordinator

| Action | Details |
|--------|---------|
| 8.1 | Issue drafts for Mechanical review (equipment data) |
| 8.2 | Issue drafts for Civil/Structural review (foundations) |
| 8.3 | Issue drafts for Electrical review (motors, cables) |
| 8.4 | Verify consistency with piping design (DEL-02.12) |
| 8.5 | Collect and consolidate coordination comments |
| 8.6 | Resolve conflicts through coordination meetings |
| 8.7 | Incorporate agreed changes |

**Output:** Coordinated drawings, meeting notes

---

### Step 9: Internal Quality Check
**Responsible:** Piping Lead, Piping Designer

| Action | Details |
|--------|---------|
| 9.1 | Self-check against Specification requirements |
| 9.2 | Verify equipment data matches datasheets |
| 9.3 | Verify nozzle schedules complete |
| 9.4 | Verify consistency with 3D model |
| 9.5 | Check CAD standards compliance |
| 9.6 | Discipline check by Piping Lead |
| 9.7 | Document and resolve check comments |

**Output:** Checked drawings, resolved comment log

---

### Step 10: Issue for Review (IFR)
**Responsible:** Piping Lead, Document Controller

| Action | Details |
|--------|---------|
| 10.1 | Finalize drawings for IFR issue |
| 10.2 | Assign official document numbers per DEL-00.02 |
| 10.3 | Complete transmittal documentation |
| 10.4 | Submit to TM for review per SOW requirements |
| 10.5 | Log issue in MDR (DEL-00.01) |

**Output:** IFR drawings issued, transmittal record

---

### Step 11: Comment Resolution
**Responsible:** Piping Designer/Engineer

| Action | Details |
|--------|---------|
| 11.1 | Receive and log TM review comments |
| 11.2 | Review and categorize comments |
| 11.3 | Prepare responses/resolutions for each comment |
| 11.4 | Coordinate clarifications with TM as needed |
| 11.5 | Implement accepted changes in drawings |
| 11.6 | Document comment resolution in log |

**Output:** Comment response log, updated drawings

---

### Step 12: Issue for Design (IFD)
**Responsible:** Piping Lead, Document Controller

| Action | Details |
|--------|---------|
| 12.1 | Incorporate all resolved comments |
| 12.2 | Final quality check |
| 12.3 | Update revision block and issue status |
| 12.4 | Issue IFD drawings |
| 12.5 | Update MDR |

**Output:** IFD drawings issued

---

### Step 13: Vendor Data Update Cycle
**Responsible:** Piping Designer/Engineer, Mechanical Lead

| Action | Details |
|--------|---------|
| 13.1 | Monitor vendor data receipt status |
| 13.2 | Review certified vendor drawings when received |
| 13.3 | Update equipment outlines to match vendor data |
| 13.4 | Update nozzle schedules to match vendor data |
| 13.5 | Update anchor bolt patterns to match vendor data |
| 13.6 | Update 3D model with vendor data |
| 13.7 | Re-coordinate with affected disciplines |
| 13.8 | Track vendor data incorporation status |

**Output:** Updated drawings with vendor data

---

### Step 14: Progress to IFC
**Responsible:** Piping Designer/Engineer

| Action | Details |
|--------|---------|
| 14.1 | Verify all vendor data incorporated |
| 14.2 | Incorporate HAZOP action items as applicable |
| 14.3 | Final coordination check |
| 14.4 | Final quality check |
| 14.5 | Issue IFC drawings |
| 14.6 | Update MDR |

**Output:** IFC drawings issued

---

### Step 15: Native File Submission
**Responsible:** CAD Technician, Document Controller

| Action | Details |
|--------|---------|
| 15.1 | Prepare native CAD files per SOW-0259 requirements |
| 15.2 | Verify file naming convention compliance |
| 15.3 | Submit native files with PDF packages |
| 15.4 | Confirm receipt by TM document control |

**Output:** Native files submitted
