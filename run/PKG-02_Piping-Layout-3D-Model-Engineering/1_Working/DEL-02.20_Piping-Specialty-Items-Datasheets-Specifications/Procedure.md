## 1. Purpose

This procedure defines the step-by-step process for developing Piping Specialty Item Datasheets and Specifications for the Puget Sound Optimization Project (PSO).

---

## 2. Scope

This procedure applies to the development of datasheets, specifications, and registers for all piping specialty items within the PSO scope at Sumas Tank Farm.

## 5. Procedure

### Step 1: Identify Specialty Items

**1.1** Review P&IDs systematically:
- Identify all specialty symbols (strainers, special valves, etc.)
- Review P&ID notes for special requirements
- Cross-reference with equipment list

**1.2** Review other sources:
- Process flow diagrams for specialty equipment
- Valve list for special valve types
- Relief study for relief devices
- Safety study for safety devices

**1.3** Compile specialty item list:
- Item type and description
- P&ID reference
- Service description
- Preliminary size

**1.4** Review list with Piping Lead and Process Lead.

**Deliverable:** Preliminary specialty item list

---

### Step 2: Establish Specialty Item Register

**2.1** Create register template with columns:
- Tag number
- Description
- P&ID reference
- Datasheet number
- Size/rating
- Vendor/manufacturer
- Requisition reference
- Status

**2.2** Assign tag numbers:
- Follow TM tagging convention (TBD)
- Ensure unique tags within project
- Group by item type

**2.3** Populate register with identified items.

**2.4** Submit register for Piping Lead approval.

**Deliverable:** Approved specialty item register

---

### Step 3: Obtain Design Inputs

**3.1** Obtain design conditions from DBM (DEL-01.01):
- Design pressure
- Design temperature
- Operating conditions
- Fluid properties

**3.2** Obtain sizing inputs from Process:
- Flow rates
- Pressure drops (allowable)
- Performance requirements

**3.3** Obtain material requirements from PMC Index (DEL-02.25):
- Body materials
- Trim materials
- Seal materials

**3.4** Document all inputs for traceability.

**Deliverable:** Design input documentation

---

### Step 4: Perform Sizing/Selection

**4.1** For each specialty item requiring sizing:
- Review manufacturer catalogs
- Perform sizing calculations
- Document calculation basis

**4.2** Select appropriate model/type:
- Consider technical requirements
- Consider availability and lead time
- Consider cost factors

**4.3** Document selection rationale.

**4.4** File calculations in project documentation.

**Deliverable:** Sizing calculations and selection documentation

---

### Step 5: Develop Datasheets

**5.1** Obtain datasheet template:
- Use TM template if available
- Use project standard template
- Use industry standard format

**5.2** Complete datasheet sections:

| Section | Content |
|---------|---------|
| General | Tag, service, P&ID reference |
| Design Conditions | Pressure, temperature, flow |
| Materials | Body, trim, seals per PMC |
| Dimensions | Size, rating, connections |
| Performance | Cv, DP, capacity |
| Accessories | Indicators, actuators |
| Testing | Shop test, field test |
| Notes | Special requirements |

**5.3** Complete one datasheet per specialty item.

**5.4** Cross-reference datasheets in register.

**Deliverable:** Draft specialty item datasheets

---

### Step 6: Internal Review

**6.1** Perform self-check:
- All required fields completed
- Design conditions correct
- Materials per PMC
- Codes referenced

**6.2** Submit to Checker for review:
- Technical accuracy
- Consistency with P&IDs
- Material compliance
- Completeness

**6.3** Address review comments.

**6.4** Obtain discipline lead approval.

**Deliverable:** Reviewed datasheets with comment resolution

---

### Step 7: Cross-Discipline Coordination

**7.1** Coordinate with Process:
- Verify design conditions
- Confirm sizing adequacy
- Review performance requirements

**7.2** Coordinate with Stress (if applicable):
- Expansion joint movements
- Special support requirements

**7.3** Coordinate with Instrumentation (if applicable):
- Control valve interfaces
- Instrument connections

**7.4** Resolve all coordination comments.

**Deliverable:** Coordinated datasheets

---

### Step 8: Issue for Review (IFR)

**8.1** Prepare datasheet package for IFR:
- All datasheets in PDF format
- Updated specialty item register
- Transmittal documentation

**8.2** Submit to Document Control for issue to TM.

**8.3** Track TM comments and prepare responses.

**Deliverable:** IFR datasheet package

---

### Step 9: Comment Resolution and IFD Issue

**9.1** Receive and log TM review comments.

**9.2** Prepare comment responses:
- Accept and incorporate
- Reject with justification
- Clarify as needed

**9.3** Update datasheets per accepted comments.

**9.4** Issue as IFD at end of FEED.

**Deliverable:** IFD datasheet package

---

### Step 10: Support Procurement

**10.1** Prepare material requisitions from datasheets:
- Include datasheet as MR attachment
- Specify vendor data requirements
- Include drawing review requirements

**10.2** Support vendor bid evaluation:
- Review technical compliance
- Clarify requirements as needed

**10.3** Update register with procurement status.

**Deliverable:** Material requisitions, bid evaluations

---

### Step 11: Review Manufacturer Drawings

**11.1** Receive vendor data packages:
- Certified drawings
- Data sheets
- Installation instructions
- Spare parts lists

**11.2** Review vendor drawings:
- Verify compliance with datasheet
- Check dimensions against layout
- Verify materials and ratings
- Check testing provisions

**11.3** Prepare review response:
- Code 1: Approved
- Code 2: Approved with comments
- Code 3: Revise and resubmit
- Code 4: Rejected

**11.4** Return response via procurement.

**11.5** Track resubmissions until final approval.

**Deliverable:** Vendor drawing reviews and endorsements

---

### Step 12: Detailed Design Updates

**12.1** Update datasheets during detailed design:
- Incorporate vendor data
- Finalize specifications
- Resolve all TBD items

**12.2** Maintain register throughout project:
- Update status
- Track procurement
- Add new items as identified

**12.3** Issue progressive revisions as required.

**Deliverable:** Updated datasheets and register

---

### Step 13: Issue for Construction (IFC)

**13.1** Finalize all datasheets:
- All TBD items resolved
- Vendor data incorporated
- All review comments closed

**13.2** Perform final review and obtain approvals.

**13.3** Issue as IFC through Document Control:
- PDF versions of all datasheets
- Final specialty item register
- Native files as required

**13.4** Archive per SOW requirements.

**Deliverable:** IFC datasheet package
