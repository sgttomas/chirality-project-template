## 1. Purpose

### 1.1 Objective

This procedure defines the step-by-step process for developing, approving, and implementing the Deliverable Review & Verification Plan for the Puget Sound Optimization Project (PSO).

**Note:** Terminal Manager is abbreviated as "TM" throughout this document. Interdisciplinary Review is abbreviated as "IDR".

### 1.2 Scope

This procedure covers:
- Development of the review and verification plan
- Terminal Manager (TM) review and agreement process
- Implementation and training
- Execution of internal review activities
- Management of TM review cycles

## 3. Steps

### 3.1 Plan Development

| Step | Action | Responsible | Output |
|------|--------|-------------|--------|
| 3.1.1 | Review SOW QA/design control requirements (Section 4.2.4) | QA Lead | Requirements summary |

| 3.1.3 | Obtain deliverable list from Master Document Register (MDR) structure | QA Lead | Deliverable types |

| 3.1.5 | Define roles and responsibilities (originator, checker, reviewer, approver, interdisciplinary reviewer per Spec RVP-RP-002) | QA Lead | RACI matrix |
| 3.1.6 | Define verification methods by deliverable type (calculations, drawings, models, vendor data per Spec RVP-VR-001 to RVP-VR-005) | QA Lead + Discipline Leads | Verification methods |
| 3.1.7 | Define comment tracking and resolution process (per Spec RVP-RP-006) | QA Lead | Comment process |
| 3.1.8 | Define documentation and record requirements | QA Lead | Documentation requirements |
| 3.1.9 | Compile complete plan document | QA Lead | Draft plan |
| 3.1.10 | Conduct internal review | QA Manager | Reviewed draft |

### 3.5 Terminal Manager (TM) Review Cycle Management

| Step | Action | Responsible | Output |
|------|--------|-------------|--------|
| 3.5.1 | Submit deliverable package to TM as Issued for Review (IFR) (per Spec RVP-RP-004, RVP-RP-005) | Document Control | Transmittal |
| 3.5.2 | Receive TM comments | Document Control | Comment log |
| 3.5.3 | Distribute comments to discipline leads | Document Control | Distribution record |
| 3.5.4 | Prepare responses per comment format (per Spec RVP-RP-006) | Discipline Leads | Draft responses |
| 3.5.5 | QA review of responses | QA Lead | Verified responses |
| 3.5.6 | Implement accepted changes | Originators | Revised deliverables |
| 3.5.7 | Submit revised deliverables and responses | Document Control | Transmittal |
| 3.5.8 | Repeat until TM accepts (status to Issued for Design (IFD) / Issued for Construction (IFC)) | All | Final deliverables |

## 4. Verification

### 4.1 Quality Control Checklist

| Check Item | Verification Method | Pass Criteria | Spec Ref |
|------------|---------------------|---------------|----------|
| All deliverable types covered | Matrix review | All types listed with review levels | RVP-RP-001 |
| Roles assigned | RACI review | All roles (originator, checker, reviewer, approver, interdisciplinary reviewer) filled with named personnel | RVP-RP-002 |
| Comment log current | Log inspection | All comments logged and tracked per format | RVP-RP-006 |
| Review records complete | Audit | Sign-offs present for issued deliverables | RVP-DR-001 |
| TM comments addressed | Response review | All comments have responses | RVP-RP-005 |
| Verification methods defined | Plan review | Methods defined for calculations, drawings, models, vendor data, consistency | RVP-VR-001 to RVP-VR-005 |

### 4.2 Review Requirements

| Milestone | Reviewer | Approval |
|-----------|----------|----------|
| Draft Complete | QA Manager | Internal |
| TM Submission | Terminal Manager (TM) Project Team | TM |
| Process Updates | QA Manager | Internal |

### 4.3 Issue Resolution

| Issue Type | Resolution Process |
|------------|---------------------|
| Review bottleneck | Escalate to Project Manager; reassign resources |
| Comment disagreement | Escalate through technical leads; Terminal Manager (TM) decision if unresolved |
| Missed review | Conduct review before next submission; log in Non-Conformance Report (NCR) if delivered |
