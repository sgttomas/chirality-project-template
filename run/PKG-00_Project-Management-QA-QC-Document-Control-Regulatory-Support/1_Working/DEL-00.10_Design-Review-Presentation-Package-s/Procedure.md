# Procedure: DEL-00.10 Design Review Presentation Package(s)

## 1. Purpose
This procedure defines the steps for planning, preparing, conducting, and following up on design reviews for the PSO project.

## 2. Scope
Applies to all formal design reviews conducted during FEED and Detailed Design phases for the PSO project at Sumas Tank Farm.

## 3. Responsibilities

| Role | Responsibilities |
|------|------------------|
| Project Manager | Chair reviews; approve packages; ensure follow-up |
| Design Review Coordinator | Plan and coordinate reviews; compile packages; manage actions |
| Discipline Leads | Prepare discipline content; present at reviews; resolve discipline actions |
| Risk Coordinator | Provide risk and LP/HF status inputs |
| Interface Manager | Provide interface status inputs |
| Document Controller | Manage package submission; file records |
| TM Project Lead | Participate in reviews; approve design progression |

## 4. Procedure Steps

### 4.1 Review Planning
| Step | Action | Responsible | Output |
|------|--------|-------------|--------|
| 4.1.1 | Confirm review milestone date per project schedule | Project Manager | Review date |
| 4.1.2 | Assess design readiness against review criteria | Discipline Leads | Readiness assessment |
| 4.1.3 | Confirm review scope (disciplines, systems to cover) | Project Manager | Review scope |
| 4.1.4 | Identify required attendees (EC, TM, MAC, others) | Design Review Coordinator | Attendee list |
| 4.1.5 | Reserve meeting facility/virtual platform | Design Review Coordinator | Meeting logistics |
| 4.1.6 | Distribute meeting invitation and agenda | Design Review Coordinator | Meeting invitation |
| 4.1.7 | Confirm TM participation | Project Manager | TM confirmation |

### 4.2 Package Preparation - Discipline Content
| Step | Action | Responsible | Output |
|------|--------|-------------|--------|
| 4.2.1 | Process Engineering prepares process/layout status | Process Lead | Process section |
| 4.2.2 | Piping Engineering prepares piping/layout status | Piping Lead | Piping section |
| 4.2.3 | Mechanical Engineering prepares equipment status | Mechanical Lead | Mechanical section |
| 4.2.4 | Civil/Structural prepares foundations/structural status | Civil Lead | Civil section |
| 4.2.5 | Electrical Engineering prepares electrical status | Electrical Lead | Electrical section |
| 4.2.6 | I&C Engineering prepares control system status | I&C Lead | I&C section |
| 4.2.7 | Architectural prepares building design status | Architectural Lead | Architectural section |

### 4.3 Package Preparation - Cross-Cutting Content
| Step | Action | Responsible | Output |
|------|--------|-------------|--------|
| 4.3.1 | Prepare project status overview (schedule, budget) | Project Manager | Status overview |
| 4.3.2 | Compile interface status summary | Interface Manager | Interface status |
| 4.3.3 | Compile risk status summary | Risk Coordinator | Risk status |
| 4.3.4 | Compile LP/HF action status | Risk Coordinator | LP/HF status |
| 4.3.5 | Compile HAZOP action status | Process Lead | HAZOP status |
| 4.3.6 | Compile prior review action status | Design Review Coordinator | Action status |
| 4.3.7 | Prepare constructability status (if applicable) | Construction Lead | Constructability status |
| 4.3.8 | Compile vendor data integration status | Mechanical Lead | Vendor data status |

### 4.4 Package Assembly and Review
| Step | Action | Responsible | Output |
|------|--------|-------------|--------|
| 4.4.1 | Compile all sections into presentation package | Design Review Coordinator | Draft package |
| 4.4.2 | Compile supporting drawings and documents | Design Review Coordinator | Supporting materials |
| 4.4.3 | Internal review of draft package | Project Manager | Review comments |
| 4.4.4 | Incorporate comments and finalize package | Design Review Coordinator | Final package |
| 4.4.5 | Distribute package to attendees (3-5 days before) | Design Review Coordinator | Distribution record |

### 4.5 Design Review Meeting Execution
| Step | Action | Responsible | Output |
|------|--------|-------------|--------|
| 4.5.1 | Open meeting; confirm attendance and objectives | Project Manager | Meeting start |
| 4.5.2 | Present project status overview | Project Manager | Status presented |
| 4.5.3 | Present discipline status (each discipline) | Discipline Leads | Disciplines presented |
| 4.5.4 | Present interface status | Interface Manager | Interfaces presented |
| 4.5.5 | Present risk and LP/HF status | Risk Coordinator | Risk/LP/HF presented |
| 4.5.6 | Present prior action status | Design Review Coordinator | Actions presented |
| 4.5.7 | Facilitate discussion and Q&A | Project Manager | Discussion notes |
| 4.5.8 | Capture new actions with owners and due dates | Note Taker | Action list |
| 4.5.9 | Confirm review outcome (approve/hold/revise) | Project Manager + TM | Decision record |
| 4.5.10 | Summarize actions and next steps | Project Manager | Meeting close |

### 4.6 Post-Review Documentation
| Step | Action | Responsible | Output |
|------|--------|-------------|--------|
| 4.6.1 | Prepare draft meeting minutes | Note Taker | Draft minutes |
| 4.6.2 | Circulate draft minutes for comment | Design Review Coordinator | Comment request |
| 4.6.3 | Incorporate comments and finalize minutes | Note Taker | Final minutes |
| 4.6.4 | Distribute final minutes to all attendees | Design Review Coordinator | Distribution record |
| 4.6.5 | Submit package and minutes to document control | Document Controller | Submission record |
| 4.6.6 | Archive supporting materials | Document Controller | Archive record |

### 4.7 Action Management
| Step | Action | Responsible | Output |
|------|--------|-------------|--------|
| 4.7.1 | Enter all actions in tracking system | Design Review Coordinator | Action register |
| 4.7.2 | Notify action owners of assignments | Design Review Coordinator | Notification record |
| 4.7.3 | Action owners complete assigned work | Action Owners | Completion work |
| 4.7.4 | Update action status regularly | Design Review Coordinator | Updated register |
| 4.7.5 | Report action status at project meetings | Design Review Coordinator | Status reports |
| 4.7.6 | Escalate overdue or blocked actions | Design Review Coordinator | Escalation record |
| 4.7.7 | Verify action closure and update register | Design Review Coordinator | Closed actions |

### 4.8 Escalation Process
| Step | Action | Responsible | Trigger |
|------|--------|-------------|---------|
| 4.8.1 | Identify action requiring escalation | Design Review Coordinator | Overdue, blocked |
| 4.8.2 | Document issue and attempted resolution | Design Review Coordinator | Escalation memo |
| 4.8.3 | Present to Project Manager | Design Review Coordinator | PM decision |
| 4.8.4 | Escalate to TM if unresolved | Project Manager | TM notification |
| 4.8.5 | Document resolution | Design Review Coordinator | Decision record |
