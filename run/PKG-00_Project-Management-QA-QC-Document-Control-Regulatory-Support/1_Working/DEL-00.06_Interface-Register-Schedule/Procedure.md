## 1. Purpose
This procedure defines the steps for creating, maintaining, and using the Interface Register & Schedule throughout the PSO project.

## 2. Scope
Applies to all interfaces requiring coordination between:
- TM and Engineering Contractor
- Engineering Contractor and MAC
- Engineering Contractor and other contractors
- Internal discipline-to-discipline coordination
- Vendor interfaces

## 3. Responsibilities

| Role | Responsibilities |
|------|------------------|
| Interface Coordinator | Maintain register, facilitate resolution, report status |
| Discipline Leads | Identify interfaces, assign owners, track discipline items |
| Project Manager | Review status, escalate issues, approve closures |
| Interface Owners | Resolve assigned interfaces within due dates |

## 4. Procedure Steps

### 4.1 Register Initialization
| Step | Action | Responsible | Output |
|------|--------|-------------|--------|
| 4.1.1 | Obtain or create register template per TM requirements | Interface Coordinator | Blank register template |
| 4.1.2 | Conduct initial interface identification workshop | Interface Coordinator + Discipline Leads | Initial interface list |
| 4.1.3 | Assign interface owners and due dates | Project Manager | Populated register |
| 4.1.4 | Distribute register to authorized parties (TM Project Team, Engineering Contractor leads, MAC, and other interface-related parties per Specification SPEC-06.06) | Interface Coordinator | Register distribution record |

### 4.2 Interface Entry Creation
| Step | Action | Responsible | Output |
|------|--------|-------------|--------|
| 4.2.1 | Identify new interface requiring coordination | Any team member | Interface notification |
| 4.2.2 | Assign Interface ID per naming convention (INT-[Category]-[Sequential], see Specification SPEC-06.13) | Interface Coordinator | Unique ID assigned |
| 4.2.3 | Complete required fields (description, type, parties, Interface Owner) per Datasheet schema | Interface Coordinator | Draft entry |
| 4.2.4 | Confirm Interface Owner assignment with responsible party | Interface Coordinator | Confirmed ownership |
| 4.2.5 | Establish due date aligned with project schedule milestones | Interface Coordinator + Interface Owner | Due date set |
| 4.2.6 | Add entry to register | Interface Coordinator | Updated register |

### 4.3 Interface Tracking and Updates
| Step | Action | Responsible | Output |
|------|--------|-------------|--------|
| 4.3.1 | Review open interfaces weekly (minimum) | Interface Coordinator | Status assessment |
| 4.3.2 | Contact Interface Owners of at-risk or overdue items | Interface Coordinator | Status update or escalation |
| 4.3.3 | Update status and notes in register (log date and modifier per Specification SPEC-06.31) | Interface Coordinator | Updated register |
| 4.3.4 | Report interface status at coordination meetings | Interface Coordinator | Meeting record |

### 4.4 Interface Resolution and Closure
| Step | Action | Responsible | Output |
|------|--------|-------------|--------|
| 4.4.1 | Confirm interface is fully resolved | Interface Owner | Resolution confirmation |
| 4.4.2 | Document resolution approach and outcome | Interface Owner | Resolution notes |
| 4.4.3 | Obtain closure approval (if required) | Project Manager | Approval record |
| 4.4.4 | Update register status to Closed | Interface Coordinator | Closed entry |
| 4.4.5 | Archive supporting documentation | Interface Coordinator | Documentation filed |

### 4.5 Escalation Process (ASSUMPTION)
| Step | Action | Responsible | Trigger |
|------|--------|-------------|---------|
| 4.5.1 | Notify Discipline Lead | Interface Coordinator | < 1 week overdue |
| 4.5.2 | Escalate to Project Manager | Discipline Lead | 1-2 weeks overdue |
| 4.5.3 | Escalate to TM Project Lead | Project Manager | > 2 weeks overdue |
| 4.5.4 | Convene resolution meeting | TM Project Lead | Critical interface impacted |

## 5. Register Maintenance Schedule

| Activity | Frequency | Responsible |
|----------|-----------|-------------|
| Update interface status | Weekly (minimum) | Interface Coordinator |
| Review open interfaces | Weekly coordination meeting | Project Team |
| Report to TM | As defined in reporting requirements | Project Manager |
| Archive closed interfaces | Monthly | Interface Coordinator |
| Full register audit | Quarterly | QA/QC Lead |
